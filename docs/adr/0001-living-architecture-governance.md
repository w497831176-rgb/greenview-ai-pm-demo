# ADR-0001｜V1.8 使用 Living Architecture 治理

- 状态：Accepted
- 日期：2026-07-18
- 决策范围：V1.8 顶层设计、后续实现与 Codex 记忆

## 背景

V1.8 需要防止长会话、多人或多代理编码造成无记录的架构漂移，但顶层设计也会随着产品目标、运行证据和技术条件变化而修订。把架构文档理解为永久不可修改，会阻碍合理演进；只依赖聊天上下文，又会让关键决策被遗忘。

## 决策

`08-v1.8-enterprise-runtime-architecture-contract.md` 采用 Living Architecture：

- `Active` 版本是当前实现参考；
- 允许提出有证据的修订；
- 修订必须记录影响和替代方案，更新 ADR、文档版本、验收契约及替代关系；
- 新版本批准后替代旧版本；
- 实现不得在没有修订记录的情况下偏离当前版本；
- Codex 记忆只保存最新文档入口，不复制架构正文。

## 结果

这既保留工程约束和可追溯性，也允许架构在证据驱动下持续优化。旧版本继续保留用于解释历史决策，但不永久约束后续版本。
