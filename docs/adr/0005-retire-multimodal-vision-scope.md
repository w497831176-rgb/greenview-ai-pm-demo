# ADR-0005｜退役多模态 Vision 演示范围

- 状态：Accepted
- 日期：2026-07-23
- 决策范围：图片上传、Kimi Vision、图片分析上下文与多模态演示
- 决策人：项目所有者

## 背景

YIAI物业是 AI 产品经理面试演示，不是生产物业系统。当前四个产品目标集中在：

1. Agent、Skill、RAG、MCP、人机协同、Evaluation/Badcase、Trace 与成本的真实命中；
2. Agent、Skill、MCP、RAG 的动态新增与下一新会话生效；
3. Badcase/Evaluation 优化闭环；
4. Token、模型与成本优化的可解释治理。

图片辅助报修需要独立上传 UI、图片资产存储、Kimi Provider、Vision/OCR 结构化结果、
聊天上下文注入和额外失败处理。它不属于上述四个目标的必要证据，却显著增加依赖、
接口、数据和演示讲解复杂度。

## 决策

1. 从当前产品范围退役全部多模态 Vision 功能。
2. 删除图片上传、预览、Kimi 分析和结果展示的前端入口。
3. 删除 `/api/multimodal/*` 路由及实现。
4. 删除聊天请求中的 `image_analysis_ids`，V1.7 回滚适配器也不再注入图片分析上下文。
5. 删除 Compose 中的 Kimi 环境变量。
6. 不删除已有 `multimodal_assets`、`multimodal_analysis_runs` 表或历史资产文件；它们作为
   不再被应用访问的历史数据保留，避免破坏 NAS 数据卷。
7. 不因本次退役盲目删除 `torch`、`Pillow` 等共享依赖；它们仍可能被 RAG 重排、
   文档解析或依赖树使用。依赖瘦身另行评估。
8. 增加无模型契约，防止隐藏 API、请求字段或前端入口被无意恢复。

## 结果

- 聊天恢复纯文本输入，主链少一次外部 Provider 和资产写入。
- 四目标架构、验收矩阵和面试演示不再包含多模态。
- 现有业务数据不受破坏。
- 若未来重新引入 Vision，必须以新的产品目标、独立 ADR、成本与验收契约重新立项，
  不能直接恢复旧模块。

## 回滚

如确需恢复，必须从退役前 Git 版本重新实现并重新评审，而不是重新挂载历史数据库表。
