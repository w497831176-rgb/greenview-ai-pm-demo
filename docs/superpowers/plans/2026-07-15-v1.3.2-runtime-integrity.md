## Task 2: RAG + MCP Composite Retrieval

**Files:**
- Modify: `rag_retrieval.py:272-330` (`advanced_search`), `app/chat.py:386-432` (`_build_rag_context`)
- Test: `scripts/test_knowledge_mcp_hygiene.py` or new regression script with the exact long query

- [ ] **Step 2.1: Capture baseline with debug endpoint**
  - POST the exact composite question to `/api/retrieval/debug` and record `keyword_results`, `semantic_results`, `fused_results`, `results`, `context_score`, and why citations are empty.

- [ ] **Step 2.2: Implement composite retrieval in `advanced_search`**
  - Parse the query for `《title》` patterns; for each title, run a title-boosted search (e.g., query = title) and include matching doc chunks.
  - Split the query into sub-questions (sentences ending with `？`/`?`) and run `advanced_search` for each with the same settings.
  - Run the original full-query search.
  - Merge all candidate results by `(doc_id, chunk_index)`, keeping the highest fused score.
  - Apply `context_threshold` per individual query path; only filter on context_score when the path itself produced it.
  - If and only if all paths return no results, return empty and allow caller to create auto badcase.

- [ ] **Step 2.3: Ensure citations survive**
  - `_build_rag_context` must return citations with `doc_id`, `doc_title`, `chunk_index`, `content`, `score`.
  - Agent prompt/instructions must not permit saying "未从知识库检索到" when citations are non-empty.

- [ ] **Step 2.4: Regression test**
  - Assert route = maintenance, Skills include work-order, MCP calls include weather + work-order count/list, citations include both target docs and the FAQ "家里漏水了怎么办" chunk, and `auto_badcase_id` is null in SSE done.
