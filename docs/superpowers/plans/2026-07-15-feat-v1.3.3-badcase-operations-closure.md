**Tech Stack:** Python FastAPI + SQLite + Agno + DeepSeek API, vanilla JS frontend, Playwright, requests.

- [ ] **Step 3.2: 重构 feedback 端点**

```python
@router.post("/feedback")
async def chat_feedback(req: FeedbackRequest):
    msg = get_chat_message(req.session_id, req.message_id) if req.message_id else None
    original_query = ""
    ai_response = ""
    context_json = {}
    trace_id = None
    if msg:
        original_query = msg.get("user_message") or ""
        ai_response = msg.get("content") or ""
        trace_id = msg.get("trace_id")
        context_json = {
            "route_intent": msg.get("route_intent"),
            "route_reason": msg.get("route_reason"),
            "current_agent": msg.get("current_agent"),
            "activated_skills": msg.get("activated_skills"),
            "citations": msg.get("citations"),
            "tool_calls": msg.get("tool_calls"),
            "mcp_calls": msg.get("mcp_calls"),
            "model_id": msg.get("model_id"),
            "token_count": msg.get("token_count"),
            "token_detail": msg.get("token_detail"),
            "usage_source": msg.get("usage_source"),
            "latency_ms": msg.get("latency_ms"),
            "trace_id": trace_id,
            "session_id": req.session_id,
            "message_id": req.message_id,
        }
    else:
        # Fallback to session context
        original_query = ""
        ai_response = ""
        context_json = {"session_id": req.session_id, "message_id": req.message_id}

    badcase_id = create_badcase(
        title=f"人工反馈：{req.reason[:40]}",
        description=req.reason,
        category="pending",
        status="pending",
        source="manual",
        session_id=req.session_id,
        message_id=req.message_id,
        original_query=original_query,
        ai_response=ai_response,
        feedback_reason=req.reason,
        context_json=json.dumps(context_json, ensure_ascii=False),
        trace_id=trace_id,
        priority="high" if req.type == "thumb_down" else "medium",
    )
    add_badcase_action(
        badcase_id=badcase_id,
        action_type="manual_feedback",
        action_detail=json.dumps({"reason": req.reason, "type": req.type}, ensure_ascii=False),
        status_before="pending",
        status_after="pending",
        created_by="owner",
    )
    return {"status": "ok", "badcase_id": badcase_id, "source": "manual"}
```

## Task 8: 验证关闭规则

**Files:**
- Modify: `app/badcases.py`

- [ ] **Step 8.1: verify/close 规则**

`POST /{id}/verify`:
- Require status == `verifying`.
- Require `retest_response` is not empty.
- For knowledge/skill categories, require at least one published draft or evidence that issue is fixed.
- For mcp_capability, allow close with `closed_as_capability_gap` sub-status (do not pretend fixed).
- Update status to `closed`, set `closed_at`, `verified_by`.
- Record action.

`POST /{id}/reject`:
- Require `rejected_reason` in request body.
- Update status to `rejected`, record action.

Reject close from `fixing` directly.
