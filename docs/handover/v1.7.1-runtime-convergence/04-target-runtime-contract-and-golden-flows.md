# 04｜目标运行时契约与三条黄金演示路径

## 1. 收敛目标

不是重写系统，也不是砍掉 AI 技术栈。目标是让每一种能力只在它该拥有控制权的路径上工作。

```text
                   ┌──────────────────┐
用户输入 ─────────→ │ 路径判定/权限边界 │
                   └───────┬──────────┘
                           │
      ┌────────────────────┼────────────────────┐
      │                    │                    │
  只读咨询路径         受控操作路径          动态扩展路径
 Agent/Skill/RAG/       草稿→确认→写库       新 Agent/Skill/MCP
 MCP/Trace/成本         Trace记录结果         配置→固定问题验收
```

## 2. 路径 A：只读技术栈咨询

### 目标

一次回答展示 Agent、Skill、RAG、MCP、Trace、成本，但**绝不创建或生成正式工单**。

### 权威规则

- 用户出现“仅咨询 / 不要创建 / 不提交 / 不生成”时，工作流控制器必须返回 `None`；
- Router 在动态启用 Agent 中选择一个；
- 只有该 Agent 绑定的 Skill 能注入；
- RAG 最终 citation 才能进入回答与 UI；
- 只读 MCP 可以查询天气/工单/日历；
- 每项结构化事件必须写入 Trace；
- 若 Provider usage 不全，成本展示“不可估算原因”，而非假精确金额。

### 固定验收用例 A-01

```text
我是 3-2-1201 的王先生。卫生间天花板持续滴水。
本轮仅咨询，绝不创建、生成或提交任何工单。
请：查询上海天气；查询我的最近工单和待处理数量；
依据《常见维修问题 FAQ》《物业维修服务承诺》说明应急步骤和响应时效；
最后列出工具名与实际引用文档。
```

验收：维修 Agent、维修 Skill、天气/工单只读工具、两份正确 RAG 证据、Trace/成本存在；`work_orders` 不新增；页面引用与最终 citation 元数据一致。

## 3. 路径 B：受控业务操作

### 目标

展示“模型负责理解和收集，后端控制器负责真正行动”。这才是 Agent 有手脚但不越权的证据。

### 权威规则

- 首次明确报修：仅创建/更新 `work_order_drafts`；
- 字段完整：回复“待确认草稿”；
- 只有确认词命中时，后端调用 `create_work_order`；
- 成功回复必须带真实 `work_order_id`；
- 任何模型自然语言均不能替代数据库写入结果；
- 写操作结果写入 Trace，Token 为 0 的规则工作流要明确标示“未调用模型”。

### 固定验收用例 B-01

第一轮：`厨房漏水，我要报修。`

补充字段：

```text
房号：3-2-1201
问题描述：厨房持续漏水
紧急程度：紧急
联系电话：测试手机号
预约上门时间：明天上午
```

确认：`确认提交`

验收：前两轮只有草稿，最后一轮出现真实 `WO-...`，工单列表存在同一 ID 和字段。若数据不一致，视为失败，不能以模型文案通过。

## 4. 路径 C：动态能力扩展

### 目标

证明“新增能力”真实进入运行时，而不是只多了一张后台配置卡。

### 最小动态扩展示例

1. 新增 `儿童教育Agent`：类型 vertical、启用、职责为儿童教育/托管/亲子服务咨询；
2. 新增/编辑 `孩子托管服务` 或 `zhangxuefeng-perspective` Skill；
3. 用结构化触发词保存，例如：`儿童教育、孩子学习、不爱学习、学习动力、张雪峰`；
4. 绑定该 Skill 给儿童教育 Agent；
5. 可选绑定一个只读课程/活动 MCP（若未真实接入，明确展示为规划，不假装有）；
6. 新建会话提问：`我家孩子不爱学习怎么办？`；
7. 查看：路由到儿童教育 Agent、Skill 命中原因、工具/知识证据、Trace、成本。

### 验收边界

- 若没有课程知识库或真实课程 Tool，回答必须说“可提供方法框架/建议转人工”，不能编造具体课程与电话号码。
- “动态”验证的重点是运行时路由与 Skill 注入，不是让任何领域凭空拥有完整业务数据。

## 5. 引用契约

必须把“模型回答”“citation 元数据”“UI 点击弹窗”锁成同一份结构化数据：

1. RAG 产生最终选中 `citations[]`，每项含 doc_id、title、chunk_index、supporting text；
2. 模型只能收到这些候选证据及稳定编号；
3. 后端解析/验证回答中的编号；
4. UI 根据 `citations[]` 渲染可点击卡片，而非猜测模型文本中的任意“引用2”；
5. 未在最终回答用到的候选只能放在 Trace 调试区，不能混入“回答实际引用”。

## 6. 成本契约

每次模型调用必须记录：

`stage + model_id + usage_source + token fields + price snapshot + formula + estimated cost or unavailable reason`。

优化策略展示至少包括：

- COST-01：Flash 处理业主常规路径，Pro 限制在 A/B、Darwin 等；
- COST-02：RAG Top-K / rerank / threshold 控制输入规模；
- COST-03：Skill 有按需触发和注入字数预算；
- COST-04：历史轮数和 MCP 结果以摘要进入上下文，避免重复注入。
