## V1.2 已完成

V1.2 聚焦「AI 智能客服与工单协同」完整业务闭环演示，不扩展生产级能力。

- **Router + 垂直 Agent**
  - 1 个路由 Agent（维修 / 账单 / 投诉 / 客服 / 其他）
  - 4 个垂直 Agent：维修、账单、投诉、客服
- **Skill 动态触发**
  - 基于关键词 / Jaccard 相似度的轻量触发
  - 仅向当前路由 Agent 注入已绑定 Skill
- **RAG 可解释检索**
  - 四阶段检索可见：keyword / semantic / RRF fusion / final
  - 引用携带 doc_id、doc_title、chunk_index、原文片段、score
  - Top-K 与检索权重前后端一致
- **MCP 真实工具调用**
  - 仅保留 3 个正式 MCP Server：weather-server、workorder-server、calendar-server
  - SSE 中展示真实 tool_calls 名称与参数
- **人工接管**
  - 业主侧可请求人工
  - 员工侧可查看会话历史、接管并回复
- **Flash / Pro 模型分层**
  - 业主聊天默认 deepseek-v4-flash（thinking_enabled）
  - 复杂任务 / Darwin 使用 deepseek-v4-pro

## 已人工验证

- 天气 MCP：维修场景调用 `get_current_weather`
- 工单 MCP：维修场景调用 `count_work_orders` / `list_recent`
- 日历 MCP：日期相关查询调用 `get_current_datetime`
- RAG 分片引用：点击引用弹出精确 chunk 内容
- Top-K 生效：Top-K=1 时 citations ≤1，改回 5 后恢复
