# V1.8.0｜运行时收敛发布记录

## 发布目标

用一个可回滚的 Runtime Kernel 稳定证明能力真实命中、能力动态新增和成本治理，
而不是继续在 `app/chat.py` 叠加条件分支。

## 工作区实现

- `RuntimeRelease`、`RunConfigSnapshot`、`ToolPolicy` 与发布/回滚 API；
- Agno `WorkflowFactory`、`AgentFactory`、原生渐进 Skills 与
  `RuntimeCoordinator`；
- 只读咨询、受控操作、动态扩展三条路径，以及 `composite_acceptance` 组合验收；
- Agent-Skill/MCP/RAG 显式绑定和下一新会话生效；
- 动态 read MCP 白名单；动态 create/update MCP Proposal → Approval →
  ActionGateway → Receipt；delete/unknown 默认拒绝；
- 单一 Evidence Ledger、EvidenceSet 驱动的 Citation/历史点击分片、分阶段诚实成本；
- 8 条固定验收矩阵与无模型契约脚本；
- `app/chat.py` 只保留 12 行传输适配，V1.7 迁到隔离 Legacy Adapter；
- `RUNTIME_ENGINE=v17|v18` 与 RuntimeRelease 指针双回滚。

## 高危影响与回滚

- 数据库迁移只新增表，不删除或重写业务表；
- 默认运行引擎切换为 `v18`，可将 `RUNTIME_ENGINE=v17` 回退 Legacy Adapter；
- 配置发布错误可把 current 指针回滚到上一 validated Release；
- 已有会话永远保留原 Snapshot，回滚不改写历史 Trace、Citation、工单或 Receipt。
