## 问题

RC6 的 Git 导入链路只使用原生 MCP Client 完成 Tool Discovery，没有继续验证同一启动配置能否进入 Agno Agent。

Python MCP 被保存为虚拟环境内 Console Script 的绝对路径，例如：

```text
/app/data/mcp_packages/.../.yiai-venv/bin/mcp-server-calculator
```

该配置可以完成 Discovery，但 Agno 的 stdio MCP 启动器只接受 `python`、`uv`、`node` 等受支持的可执行程序。因此，一个仅要求 Router 指派 Agent、明确不调用能力的问题，也会在 Agent 初始化绑定 MCP 时失败。

## 修复

1. Python Git MCP 统一准备到项目 `.venv`。
2. 导入器生成 Agno 兼容的 `uv run --directory ... --no-sync ...` 启动规格。
3. model-native MCP 不再因“已绑定”而对每轮请求全部装配；只有当前消息命中已发布 Tool 的意图或触发词时才进入 Agno Tool Loop。
4. 没有 ToolPlan 的请求不再发送 `mcp.invoke` 进度事件。
5. 增加无模型合同，覆盖 Python 启动器兼容性以及“绑定不等于每轮启动”的消息级门控。
