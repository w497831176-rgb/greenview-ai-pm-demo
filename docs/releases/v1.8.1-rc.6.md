# V1.8.1 RC6｜MCP 平台化导入

发布日期：2026-07-24

状态：`BACKEND_SMOKE_VERIFIED / DEPLOYED / UI_PENDING`

## 发布目标

把 MCP 新增从开发者手填 `command/args/env` 的连接表单，升级为面向产品使用者的
插件导入流程：

```text
粘贴公开 Git 地址
→ 自动识别 Python / Node
→ 准备隔离依赖
→ 生成 stdio 启动配置
→ 连接并发现 Tool
→ 选择只读 / 新增写入 / 更新 / 暂不授权模板
→ 绑定垂直 Agent
→ 校验并发布 RuntimeRelease
→ 下一新会话验收
```

原始命令、参数、环境变量和完整运行契约保留在高级设置中。

## 实现范围

- 新增 `/api/mcp-servers/import-git`；
- 导入包写入持久化 `/app/data/mcp_packages`，不写入 Git 工作区；
- 支持 Python 与 Node MCP 仓库自动识别、依赖准备和入口推断；
- 导入成功即执行真实连接与 Tool Discovery；
- Tool 名称只产生用途建议，不自动获得读写授权；
- 前端增加四类 Tool 模板以及 Agent 绑定、RuntimeRelease 发布导航；
- 手动 MCP 新增与“连接并刷新 Tool”也会真实触发 Discovery；
- API 镜像加入轻量 Node 20/npm 运行时，Agno 与现有运行时权力边界不变。
