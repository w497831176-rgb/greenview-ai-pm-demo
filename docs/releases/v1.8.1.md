## 发布三｜v1.8.1-rc.3

触发原因：

- 用户手点发现，前端收到 Router Agent 转交原因后长期停留在三个加载点；
- 刷新后能看到完整历史回复，说明运行时已完成，但当前连接没有及时展示回答；
- 只读诊断确认 RC2 在垂直 Agent 流式运行期间故意缓存全部模型文本，直到
  Citation 校验完成后才一次性发送 `delta`；前端同时未展示 `progress`，也未处理
  SSE 尾包和缺失 `done` 的终止状态，因此形成“假卡死”体验。

- 模型生成片段以 `delta(provisional=true)` 实时发送并立即展示；

- `done.content`、最终 Citation、数据库历史消息和 Trace 仍使用同一权威全文；
- 前端展示 RAG、MCP、模型生成三个进度阶段；
- 前端统一处理 CRLF、流结束尾包和缺失 `done`，不再无限保留加载动画；
- 不修改 Agent/Skill/RAG/MCP 绑定、受控写路径、Goal 3/4 或 Agno 底层能力。

## 发布四｜v1.8.1-rc.4

触发原因：

- RC3 Web 镜像中的 `/usr/share/nginx/html/index.html` 权限为 `0600`；
- Nginx master 进程虽由 root 启动，但 worker 使用 `nginx` 用户，读取首页时返回
  `403 Permission denied`；
- 因故障位于 Web 容器静态文件层，内网入口和公网反向代理入口同时不可用。

修复与验证：

- `frontend/Dockerfile` 在复制静态资产后显式执行 `chmod 0644`，覆盖
  HTML、Tailwind、Marked 与 Nginx 站点配置，避免构建上下文权限再次污染镜像；
- 前端版本标识更新为 `V1.8.1-rc.4`；
- 仅重建并重启 `demo-os-web`，未重启 API；
- 容器内三个静态文件权限均为 `0644`；
- Web 宿主端口 `18005` 返回 200；
- 群晖反向代理 `https://127.0.0.1:18004/` 使用正式 Host 头返回 200；
- 群晖 LAN 地址 `https://192.168.50.123:18004/` 返回 200；
- `.env` 与 `check_db2.py` 保留原样且未 stage。

当前结论：`WEB_SERVING_VERIFIED / DEPLOYED / USER_BROWSER_RETRY_PENDING`。
